PaperMC Moderation Plugin

評判スコアで
サーバーを守る

ReputationBan は、通報とスタッフ操作をもとにプレイヤーの評判スコアを管理する PaperMC 向け Moderation プラグインです。SQLite によるデータ永続化、監査ログ、 自動BAN、外部連携に対応しています。

⬇ ダウンロード (v1.0.0) 導入手順を見る
PaperMC 対象サーバー
Java 25 必須ランタイム
SQLite ストレージ
6+ 外部連携対応
Features

主な機能

プレイヤー通報から自動BAN、監査ログ、外部連携まで、運営に必要な Moderation 機能をひとつのプラグインで提供します。

📊
評判スコア管理
プレイヤーごとに評判スコアを記録。通報承認で減点、良好な期間が続けばスコア回復。スコア履歴と変更理由を SQLite に保存します。
🚨
プレイヤー通報システム
/reportbad で通報。カテゴリ、クールダウン、最低プレイ時間条件、複数通報しきい値によるスタッフ審査フローを提供します。
🔨
自動 Profile BAN
スコアがしきい値を下回ると Paper Profile BAN API を使って自動BAN。BAN回数に応じた期間設定、unban、pardon コマンドも完備しています。
📋
監査ログ & CSV エクスポート
すべての重要操作を audit events として記録。プレイヤー別・イベント種別での絞り込みと CSV エクスポートで運営記録を管理します。
💾
バックアップ & メンテナンス
手動または自動 SQLite バックアップ、保持期間に基づくデータ cleanup、シークレットを除外した診断 ZIP（support bundle）を作成できます。
🔔
Discord 通知
任意の Webhook URL へ通報・BAN・unban などのイベントを通知。URL はログ・support bundle・監査 CSV に出力されない安全設計です。
🛡️
Bypass 保護
OP・LuckPerms bypass グループを通報・自動BAN対象外に保護。offline プレイヤーへの lookup も設定で対応し、fail-close 運用が可能です。
🩺
運用診断ツール
/rep doctor で DB・設定・連携の状態を一括確認。/rep integrations test で外部プラグイン連携の詳細診断を安全に実行します。
🔌
柔軟な外部連携
LuckPerms / CoreProtect / WorldGuard / GriefPrevention / PlaceholderAPI / DiscordSRV をすべて任意連携。未導入でも本体機能はフル稼働します。
Installation

導入手順

PaperMC 26.1.2 と Java 25 が必要です。SQLite JDBC は plugin.yml の libraries から自動ダウンロードされます。

1
サーバーを停止する
Paper サーバーを安全に停止してから作業します。
2
JAR を配置する
ReputationBan-1.0.0.jar をサーバーの plugins/ ディレクトリへ配置します。
3
一度起動して設定ファイルを生成する
サーバーを起動すると plugins/ReputationBan/config.yml と reputationban.db が自動生成されます。
4
設定を確認して再起動する
config.yml を確認・編集してからサーバーを再起動します。
5
動作確認コマンドを実行する
ゲーム内または console で以下を実行して正常起動を確認します。
/rep version → バージョン表示
/rep doctor → 診断レポート表示
6
Smoke コマンドで動作検証する
本番導入前に test サーバーで確認してください。
/rep backup before-smoke  /  /rep support bundle  /  /reports list all 10  /  /rep audit recent 10  /  /rep maintenance preview

Configuration

主な設定項目

plugins/ReputationBan/config.yml で動作をカスタマイズします。

スコア設定 (score)

initial-score
新規プレイヤーの初期スコア
max-score
スコアの上限値

評価・通報 (rating)

rating.enabled
通報によるスコア変更の有効化
rating.default-deduction
標準減点値
rating.min-unique-reports-before-deduction
自動減点に必要なユニーク通報者数
rating.report-window-days
複数通報しきい値の集計期間（日）

クールダウン (cooldowns)

cooldowns.global-report-seconds
通報間の最短間隔（秒）
cooldowns.same-target-cooldown-days
同一ターゲットへの再通報制限（日）
cooldowns.max-reports-per-day
1日あたりの通報上限

スコア閾値 (score-thresholds)

warning / watch / restricted / final-warning
各閾値でスタッフ通知が送信されます
ban
この閾値以下で自動BAN

スコア回復 (score-recovery)

score-recovery.enabled
定期回復の有効化
score-recovery.points-per-day
1日あたりの回復量
score-recovery.no-report-days-required
回復開始までに必要な通報なし期間（日）

BAN 設定 (ban)

ban.enabled
自動 Profile BAN の有効化
ban.threshold
自動BAN発動スコア
ban.durations
BAN 回数ごとの期間設定

保持期間 (retention)

audit-events-days / rejected-reports-days / score-history-days / bans-days
各データの保持日数（0 = 削除しない）

文字列長制限 (limits)

limits.max-report-reason-length
通報理由の最大文字数
limits.max-review-note-length
審査ノートの最大文字数
Integrations

外部連携

すべて 任意（softdepend） です。未導入でも ReputationBan 本体は正常稼働します。各プラグインは reflection 経由でロードされるため、起動不能になりません。

LuckPerms 任意
通報者の primary group を使って reporter weight を計算し通報補助情報として保存します。 bypass-groups に指定したグループのプレイヤーを保護対象に追加できます。 offline-lookup.enabled で未ロードのオフラインプレイヤーも保護可能。 PlaceholderAPI 向け placeholder にも使われます。
CoreProtect 任意
荒らし通報の審査補助として、通報者の周辺ブロック操作ログの短いサマリーを保存します。 rollback・restore・purge は行いません。/reports evidence で参照できます。 max-results: 0 にすると個別ログを保存しません。
WorldGuard 任意
通報者の現在地に適用されている region context（ID・priority・flag）を審査補助として保存します。 WorldEdit も必要です。region・flag の作成・変更・削除は行いません。
GriefPrevention 任意
通報者の現在地の claim 情報（有無・境界・admin claim かどうか）を審査補助として保存します。 claim・trust の作成・変更・削除は行いません。owner や trust count の記録は設定で制御します。
PlaceholderAPI 任意
他プラグインやスコアボードから評判スコア情報を参照できる読み取り専用連携です。 提供する placeholder：%reputationban_score% %reputationban_status% %reputationban_ban_count% など 10 種類。SQLite への同期問い合わせは行いません。
DiscordSRV 任意
通報者・対象プレイヤーの Discord アカウント連携状態を審査補助として保存します。 DiscordSRV の game channel へ通知も可能（デフォルト無効）。Discord ロール変更・コマンド実行は行いません。

PlaceholderAPI で利用できる Placeholder

%reputationban_score% %reputationban_max_score% %reputationban_score_percent% %reputationban_status% %reputationban_ban_count% %reputationban_false_report_count% %reputationban_report_banned% %reputationban_report_banned_until% %reputationban_last_seen% %reputationban_version%
Security

安全上の注意

シークレット情報の取り扱いに注意してください。

⚠️
Discord Webhook URL はシークレットです。
config.yml 以外の場所（GitHub Issue、チャット、スクリーンショット、review archive）に実 URL を貼らないでください。 ReputationBan は URL ではなく「設定済みかどうか」のみを表示します。
🔒
Support Bundle の安全設計
/rep support bundle は reputationban.db・WAL/SHM・server logs・Webhook URL を含まない診断 ZIP を作成します。url・token・secret・password などを含むキーは <redacted> に置換されます。
📋
Issue 報告時の注意
GitHub Issue には Discord bot token・Webhook URL・password・session・cookie・secret を貼らないでください。/rep doctor と /rep integrations の出力は安全に共有できます。
🛡️
Redacted Config
Support bundle には config-redacted.yml が含まれます。共有前に <redacted> になっていることを確認してください。
