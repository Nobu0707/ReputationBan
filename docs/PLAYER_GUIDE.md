# Player Guide

このガイドは、ReputationBan が導入されているサーバーで遊ぶプレイヤー向けの説明です。サーバーごとに設定や運営方針は異なるため、最終的な判断やルールは各サーバーの案内に従ってください。

## ReputationBanとは

ReputationBan は、通報や運営の確認をもとにプレイヤーの評判スコアを管理するためのプラグインです。

荒らし、暴言、スパム、詐欺、嫌がらせ、チート疑いなどを見つけたとき、プレイヤーは `/reportbad` で運営へ通報できます。通報内容は運営が確認し、必要に応じてスコア変更やBANなどの対応に使われます。

通報しただけで必ず相手がBANされるわけではありません。通報は運営判断の補助です。

## 評判スコアとは

評判スコアは、サーバー内での行動状況を表す数値です。初期値や最大値、どの行動でどれだけ下がるかはサーバー設定によって異なります。

スコアが下がる例です。

- 他人の建築物を壊す、盗むなどの荒らしをした場合。
- チャットで暴言、差別的発言、過度な煽りを繰り返した場合。
- 同じ内容を大量に投稿するスパムをした場合。
- アイテム取引や約束で相手をだました場合。
- 特定のプレイヤーへの嫌がらせを続けた場合。
- チートや不正ツールの疑いが運営に確認された場合。

## 通報する

通報コマンドは次の形式です。

```mcfunction
/reportbad <player> <category> <reason>
```

例です。

```mcfunction
/reportbad Steve griefing 他人の家の壁を壊していました
/reportbad Alex abusive_chat チャットで暴言を繰り返していました
```

`<player>` には通報したい相手のプレイヤー名、`<category>` には通報カテゴリ、`<reason>` には理由を書きます。

## カテゴリ例

サーバー設定によってカテゴリは変わる場合があります。標準的なカテゴリ例です。

- `griefing`: 荒らし、建築破壊、無断破壊など。
- `abusive_chat`: 暴言、攻撃的なチャットなど。
- `spam`: 同じ内容の連投や宣伝など。
- `scam`: 詐欺、取引トラブルなど。
- `harassment`: 嫌がらせ、つきまといなど。
- `cheating`: チート、不正ツール疑いなど。
- `other`: その他、分類しにくい問題です。

## 通報理由の書き方

通報理由は、運営があとで確認しやすいように具体的に書いてください。

良い通報理由の例です。

- `村の共有倉庫からダイヤを持ち出して返していません`
- `20時ごろからチャットで同じ宣伝を何度も流しています`
- `私の家の壁と床を壊していました`

悪い通報理由の例です。

- `むかつく`
- `なんとなく`
- `BANして`
- `嫌いだから`

いつ、どこで、何をしていたかが分かる理由にすると、運営が判断しやすくなります。

## 通報時の注意

- 通報しただけで必ず相手がBANされるわけではありません。
- 通報は運営判断の補助です。
- 同じ相手への連続通報には制限がある場合があります。
- 1日や1週間に通報できる回数には上限がある場合があります。
- 通報に必要な最低プレイ時間や参加日数が設定されている場合があります。
- サーバーによって設定は異なります。
- サーバーによって設定は異なるため、必ずそのサーバーのルールも確認してください。

虚偽通報や悪意ある通報を繰り返すと、通報制限を受ける場合があります。相手を困らせる目的や、証拠のない報復目的の通報はしないでください。

## 自分のスコア確認

自分の評判スコアは次のコマンドで確認できます。

```mcfunction
/rep
```

表示される内容はサーバー設定や権限によって異なります。自分以外のスコアを見られるかどうかも、サーバーの権限設定によって変わります。

## スコア回復の可能性

サーバー設定によっては、一定期間新しい通報や問題行動がない場合にスコアが少しずつ回復することがあります。

回復量、回復までに必要な日数、最大スコアはサーバーごとに異なります。詳しい条件は運営に確認してください。

## BANについて

スコアが大きく下がった場合や、運営が重大な問題と判断した場合、BANされることがあります。

BANの期間は、サーバー設定や過去のBAN回数によって変わる場合があります。ReputationBan は運営判断を補助するための仕組みであり、最終判断はサーバー運営が行います。

## 運営が確認できる証拠情報

通報時には、サーバー設定や導入プラグインに応じて、運営が次のような情報を確認できる場合があります。

- 通報者、対象プレイヤー、カテゴリ、理由、時刻。
- 通報時の周辺ブロック操作ログ。
- 通報地点の保護地域や区画情報。
- DiscordSRV のアカウント連携状態。

これらは通報内容を確認するための補助情報です。プレイヤーが個人情報やシークレットを通報理由に書く必要はありません。

## よくある質問

### 通報したらすぐBANされますか

いいえ。通報しただけで必ずBANされるわけではありません。運営が内容や証拠情報を確認します。

### 間違えて通報しました

すぐに運営へ連絡してください。サーバーによっては通報の取り消しや補足説明の方法が用意されています。

### 自分のスコアが低い理由を知りたいです

まず `/rep` で確認してください。詳しい理由や異議申し立ての方法は、サーバーの運営ルールに従ってください。

### 友達同士の冗談でも通報されますか

通報はできますが、運営が状況を確認します。相手が嫌がっている行為や、周囲に迷惑をかける行為は避けてください。

### 虚偽通報をするとどうなりますか

虚偽通報や悪意ある通報を繰り返すと、通報制限や運営対応の対象になる場合があります。

### サーバーによってルールは違いますか

はい。カテゴリ、減点、BAN条件、回復条件、通報制限はサーバーによって異なります。必ずそのサーバーのルールを確認してください。
